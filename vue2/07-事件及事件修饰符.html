Vue事件处理 ​

1. 原生 javaScript 行内事件绑定 ​

html
<style>
    .box{
        color:red;
    }
    .wrap{
        color:skyblue;
    }
</style>
<div id="app">
    <h2 id="box" class="box">Hello World</h2>
    <button onclick="changColor()">点击切换颜色</button>
</div>

<script>   
    let className = "box"
    function changColor(){
        if(className =='box'){
            box.className = className = "wrap"

        }else{
            box.className = className =  "box"
        }
    }
</script>

注意:

这种方式不常用

  1. 因为这种绑定的事件函数必须是全局的,即window下的函数

  2. onclick属性值是函数执行字符串

但是vue采用这种方式绑定事件,因为这种绑定方式比较直观的处理事件的绑定, 省去了大量获取DOM的操作,

不用担心Vue的问题, Vue在采用行内事件绑定,内部肯定做了大量的优化处理

那我们我们来好好看看Vue 是如何绑定事件的


2. Vue监听事件 ​

2.1 基本使用 ​

我们之前已经学过了v-on 指令, v-on指令就是用来处理事件的绑定, 可以简写为@

一但通过v-on指定绑定了事件,那么被绑定事件的值将不再是字符串,而是一个表达式

既然是表达式,那么我们就可以在里面做一些操作

示例:

html
<div id="app">
    <h2 class="box">点击次数: {{ count }}</h2>
    <!-- 既然click 的值是表达式,那么我们就可以动态的拿到vue ,data属性中的数据,然后操作-->
    <button v-on:click="count++">暴击</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            count: 0

        }
    })
</script>

通过限时的结果,我们能发现在我们每次触发点击事件是更新data属性中的数据. 从而触发视图的重新渲染, 记录点击次数


2.2 methods属性中定义函数 ​

但是这种操作具有局限性, 因为字符串里只能写单行表达式,如果处理复杂的程序就会有局限性, 所以我们通常使用,是把事件函数定义在vue的methods属性中, 然后在表达式中通过函数名调用事件函数

示例:

html
<!-- 限定点击次数 -->
<div id="app">
    <h2 class="box">点击次数: {{ count }}</h2>
    <button @:click="handleClick">点击</button>
</div>

<script>   
    const vm = new Vue({
        el: "#app",
        data: {
            count: 0
        },
        methods: {
            handleClick(){
                let count = this.count;
                // 限定显示最大点击次数
                this.count = Math.min(++count,5)
            }
        }
    })
</script>

通过示例了解,我们成功的将methods里的handleClick方法成功的绑定给了button的点击事件,成为了事件处理函数


2.3 方法名不能和data 数据名重名 ​

同时:我们还需要注意到的一点是, 为什么我们在表达式中都能直接获取到data中的数据和methods中的方法呢

那是因为vue 会将data中的属性和methods重点的方法全部挂在到了Vue 实例化对象vm对象上.

因为在对象上不同有同名的属性或方法, 否则就会被覆盖,

所以, methods里定义的方法名一定不能和data 中的数据名称一样, 否则会出问题, 会报错

js
// 这种写法会直接报错
// 因为data 中数据名为count  methods 中的方法名也叫count 就会报错
const vm = new Vue({
    el: "#app",
    data: {
        count: 0
    },
    methods: {
        // methods和data的数据全部会放到vm上,
       	// 而且名字不能冲突,冲突会报错,
        count(){
            let count = this.count;
            this.count = Math.min(++count,5)
        }
    }
})

其实这类似于你不能让一个对象中的属性和方法重名一样, 虽然不会报错,但是会发生覆盖,后面的同名属性的值会将前面的覆盖掉

例如:

js
let obj = {
    count: 0,
    count: function(){
        console.log("方法");

    }
}
console.log(obj);
// 打印结果
/*
{
    count: ƒ ()
    __proto__: Objec
}
*/

通过打印结果,我们发现,对象obj后面count属性的值把前面的同名属性值覆盖了,

因为Vue会将data属性中的数据和methods中的方法全部挂在到Vue实例对象上, 所以千万不能重名. 否则Vue会给你报错.


2.4 methods方法中不推荐使用箭头函数 ​

2.4.1 不推荐使用箭头函数 ​

我们之前讲过vue是以数据作为驱动的,数据的变化将会触发vue的监听,同时更改页面的渲染结果, 那也就是说我们在未来的操作中将大量操作数据, 同时的数据又是挂在在Vue 实例vm上的属性,那么我们就需要大量的通过vm对象来调用数据

在者,我们也发现methods的中的方法也是挂在在vm对象上的, 同时我们在调用方法的时候也是通过vm来调用这些方法, 所以这些方法内部的this 默认就是vm对象, 在方法中this.数据名就可以直接获取数据

也这是因为这些原因,我们不推荐方法使用箭头函数,因为这样会丢失this指向, 那么我们就不能很多的利用this 来获取数据和其他方法了.

示例:

html
<div id="app">
    <button @click="handleClick">非箭头函数中this</button>
    <button @click="handleClickTwo">箭头函数中的this</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(){
                console.log(this)  // vue实例化对象
            },
            handleClickTwo: () => {
                console.log(this) // window 对象
                
            },
        }
    })
</script>

这样我们就会发现如果我们使用普通函数, 那么要获取数据就很简单

js
handleClick(){
    console.log(this)  // vue实例化对象
    // 利用this 获取数据
    console.log(this.msg)  // 获取数据
},

如果是箭头函数,我们只能通过Vue实例化的名称来获取数据

js
handleClickTwo: () => {
    console.log(this) // window 对象
    // 因为箭头函数this是window,所以不同通过this获取数据
    // 如果此时还想获取数据,就的借助Vue实例对象vm
    console.log(vm.msg)  // vm是 vue实例化对象的标识符
},

通过对比我们就会发现,直接使用this会方便, 所以我们推荐在方法中尽量不要使用箭头函数

不是说不能,而是不建议使用箭头函数. 向较与箭头函数,ES6的方法简写方式反而来的更方便一点.

2.4.2 关于Vue方法里使用箭头函数误区 ​

但是我们需要避免一个误区:

我们推荐不使用箭头函数是methods的方法, 不是说整个Vue项目里都不能使用箭头函数,

比如我们如果在方法里还有函数,在函数中需要使用vue实例化对象我们这个时候就适合使用箭头函数

示例:

html
<div id="app">
    <button @click="handleClick">查看this</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(){
                console.log(this)  // vue实例化对象

                // 延迟修改数据
                // 1.方法内使用普通函数,查看普通函数中this
                setTimeout(function(){
                    console.log(this)  // window 对象
                    
                    // 这个是有要修改数据,还的使用vm
                    vm.msg = "wrod"
                }, 1000)

                // 2.方法内使用箭头函数, 查看函数内this
                setTimeout(() => {
                    console.log(this) // vue实例对象
                    
                    // 这个是有修改数据,依然可以使用this
                    this.msg = "wuwei"
                
                }, 2000)
            },

        }
    })
</script>

通过示例就能看出,用不用箭头函数,完全看自己的需求, 而不是说不推荐使用箭头函数,就整个Vue项目中一个箭头函数都不用了,

要关注一下是哪里不推荐使用箭头函数.


2.5 关于事件传参的问题 ​

我们会发现好像我们这种调用事件函数时没发给事件函数传递参数,那要怎么办呢

其实我们在绑定事件时说过, 事件类型属性后面是一个表达式, 那么我们在表达式里处理可以写函数名以外,还可以写函数名加括号,

示例:

html
<div id="app">
    <button @click="handleClick">不加括号的绑定事件</button>
    <button @click="handleClickTwo()">加括号的绑定事件</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(){
                console.log(this.msg)
            },
            handleClickTwo(){
                console.log(this.msg)
            },
        }
    })
</script>

我们会发现在绑定事件时无论加不加括号都可以正常处理事件,那么他们的区别在哪里呢,我们平时有该使用哪一种呢,如果不需要传参就可以使用第一种,如果需要传参就可以说会用第二种

方法传参实例

html
<div id="app">
    <button @click="handleClick('无为')">事件传参</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(name){
                console.log(`${name}向大家问好`)
            },
        }
    })
</script>

2.6 关于事件对象 ​

我们都知道,在原生的JS中,我们在绑定事件的时候可以通过事件函数的第一个参数就是时间对象,那么在Vue中会如何的,

首先我们来看不加括号的事件和加括号的事件的事件函数,第一个参数是不是事件对象

注意: 加括号并不一定会传参数

html
<div id="app">
    <button @click="handleClick">不加括号的绑定事件</button>
    <button @click="handleClickTwo()">加括号的绑定事件</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(event){
                console.log(event) // MouseEvent
            },
            handleClickTwo(event){
                console.log(event) // undefined
            },
        }
    })
</script>

我们发现, 不加括号的事件函数成功的从第一个形参中获取了事件对象,但是加括号的事件绑定却没有拿到事件对象.


那么问题来了,我们如何在需要传参的时候拿到事件对象呢?

其实如果你需要传参,那么时间对象就需要你手动的传递到事件函数中

html
<div id="app">
    <button @click="handleClickTwo($event, 18)">加括号的绑定事件</button>
</div>

<script>   
    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClickTwo(event,num){
                console.log(event)  // MouseEvent
                console.log(num)
            },
        }
    })
</script>

$event 就是事件对象, 我们需要传递参数的时候,就可以通过这种手动传参的方式将事件对象传递到事件函数中


同时我还发现, 无论你事件加不加括号,传不传参, 事件函数中的event变量都是事件对象,前提是你不能有同名的变量或形成,

如果有重名问题,就会发生覆盖

html
<div id="app">
    <button @click="handleClick">不加括号的绑定事件</button>
    <button @click="handleClickTwo($event, 18)">加括号的绑定事件</button>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClick(){
                console.log(event)   // MouseEvent
            },
            handleClickTwo(){
                console.log(event)  // MouseEvent
            },
        }
    })
</script>

通过测试我们发现都可以获取到事件对象

总结:

  1. 方法写在methods属性中
  2. 事件函数不需要传参不加括号, 需要传参加括号
  3. 事件函数无括号,默认第一个参数是事件对象
  4. 事件函数有括号,需要手动传递事件对象
  5. 无论事件函数有无括号,event变量都是事件对象,前提不能有同名变量覆盖

3. 键盘事件及键值修饰符 ​

在事件处理函数中,我们可能需要经常处理诸如阻止默认事event.preventDefault()或阻止事件冒泡event.stopPropagation() 等事情, 虽然实现起来很轻松, 但是最好的方式就是方法只用来处理纯粹的数据逻辑, 而不是去处理了DOM事件细节

为了解决这个问题，Vue.js 为 v-on 事件绑定指令提供了事件修饰符。


3.1. 修饰符的理解 ​

修饰符 (Modifiers) 是以半角句号 . 指明的特殊后缀，用于指出一个指令应该以特殊方式绑定。例如，.prevent 修饰符告诉 v-on 指令对于触发的事件调用 event.preventDefault()：

例如:

html
<div id="app">
    <form action="/wuwei" @submit.prevent="handleSubmit">
        <input type="submit" value="提交"/>
    </form>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleSubmit(){
                console.log("submit")
            }
        }
    })
</script>

默认情况下当你点击submit按钮时,页面就会发生跳转, 当我们添加了修饰符.prevent, 就会阻止默认的跳转页面的行为

那接下来就让我们看看都有哪些事件修饰符吧


3.2. 事件修饰符 ​

3.2.1. 阻止事件传播 ​

使用修饰符.stop,阻止事件的传播

html
<div @click.stop="handleClickWrap">grandson</div>

示例:

html
<div id="app">
    <div 
         class="box"
         @click="handleClickBox"

         >
        <div 
             class="wrap"
             @click.stop="handleClickWrap"
             ></div>
    </div>
</div>

<script>   

    const vm = new Vue({
        el: "#app",
        data: {
            msg: "hello"
        },
        methods: {
            handleClickBox(){
                console.log("box")
            },
            handleClickWrap(){
                console.log("wrap")
            }
        }
    })
</script>

3.2.2. 将事件绑定在捕获阶段 ​

默认的事件是绑定在冒泡阶段的,但是我们通过修饰符将事件绑定在捕获阶段

使用修饰符.capture(捕获):

html
<div @click.capture="handleClickBox">parent</div>

示例:

html
<div id="app">
    <div 
         class="box"
         @click.capture="handleClickBox"

         >
        <div 
             class="wrap"
             @click="handleClickWrap"
             ></div>
    </div>
</div>

如果将box的事件绑定在了捕获阶段,那么这里就会先执行box的事件在执行wrap的事件

阻止事件传播与事件捕获组合使用

html
<div @click.capture.stop="parent">parent</div>

示例:

html
<div id="app">
    <div 
         class="box"
         @click.capture.stop="handleClickBox"

         >
        <div 
             class="wrap"
             @click="handleClickWrap"
             ></div>
    </div>
</div>

这里事件将永远无法传递到wrap事件上, wrap的事件函数将永远不能执行, 因为每次点击wrap触发的事件流到box 就被stop终止了


3.2.3.阻止默认事件 ​

使用修饰符 .prevent

html
<a href="http://www.baidu.com" @click.prevent='child'>百度</a>

3.2.4. 事件只触发一次 ​

事件只会在第一次点击时触发,之后再点击将不再触发这个事件

使用修饰符 .once

html
<div @click.once="handleChild">child</div>

3.2.5. 当事件发生在事件源身上的时候在触发 ​

只用当事件发生在元素自己身上时才触发,

使用修饰符.self

html
<div @click.self="child">child</div>

使用修饰符时，顺序很重要；相应的代码会以同样的顺序产生。因此，用 v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。


4. 表单修饰符 ​

除了事件除了有修饰符, Vue还给其他的指令提供了一些修饰符帮我们很好的处理一些问题.

那么我们就先来看看表单修饰符


4.1 lazy修饰符 ​

使用了这个修饰符将会从“input事件”变成change事件进行同步

说白了change就是当你表单失去焦点时或者手动的点击enter键时触发,

input就是内容一变马上触发

html
<div id="app">
    <input type="text" v-model.lazy="message">
    <p>{{ message }}</p>
</div>

<script>
new Vue({
   el:"#app",
   data:{
     message:""
   }
})
</script>

4.2 number修饰符 ​

首先要明这个number并不是限制用户的输入,而是将用户输入的数据尝试绑定为 js 中的 number 类型

举个例子，如果用户输入300，data 中绑定的其实是'300'(string)，

添加 number 指令后可以得到 300(number)的绑定结果。 而如果用户输入的不是数字，这个指令并不会产生任何效果。

html
<div id="app">
    <div>
        未使用number修饰符num1
        <input type="text" v-model="num1">
    </div>
    <div>
        使用number修饰符num2
        <input type="text" v-model.number="num2">
    </div>

</div>

<script>
    new Vue({
        el:"#app",
        data:{
            num1:'',
            num2:''
        }
    })
</script>

通过示例我们发现, 同样是输入数字,未使用修饰符输入的表单得到的是字符串, 使用表单得到的是数字

注意:

如果你输入的不是数字而是字母, 就算你是用了表单number修饰符 也不会转为数字, 因为转不了,


4.3 trim 修饰符 ​

可以用来过滤前后的空格

html
<div id="app">
    <input type="text" v-model.trime="message">
    <button @click="type">获取当前的类型</button>
</div>

<script>
    new Vue({
        el:"#app",
        data:{
            message:""
        },
        methods:{
            type(){
                console.log(this.message);
            }
        }
    })
</script>

通过上面的例子我们能发现,其实表单修饰符主要作用就是在我们通过表单收集数据的时候,对数据进行先期处理, 过滤掉不需要的空格


5. 按键修饰符 ​

键盘修饰符主要修饰的是键盘事件

  • .ctrl
  • .alt
  • .shift
  • .enter
  • .tab
  • .delete (捕获“删除”和“退格”键)
  • .esc
  • .space
  • .up
  • .down
  • .left
  • .right

通常按键修饰符也可以用键盘码来代替使用

示例:

html
<!-- vue-app 是根容器  -->
<div id='vue-app'>
    <h1> 键盘 Event </h1>
    <label>姓名：</label>
    <input type="text" @keyup="logName">
    <label>年龄：</label>
    <input type="text" @keyup="logAge">
    
    <!-- 键盘修饰符 当点击enter时才会触发事件  -->
    <input type="text" @keyup.enter="logAge">
    
    <!-- 链式修饰符 当同时点击alt和enter时才会触发事件  -->
    <input type="text" @keyup.alt.enter="logAge">
</div>
<script>
//实例化Vue对象
new Vue({
    el: "#vue-app"，
    data:{
    	age: 30,
    	x:0,
    	y:0
	}，
    metholds:{
        logName: function(){
			consolog.log("你正在输入名字")
		},
        logAge: function(){
			consolog.log("你正在输入年龄")
